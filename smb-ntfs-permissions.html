SMB Shares and NTFS Permissions Lab

This lab demonstrates creating a shared folder, configuring share and NTFS permissions, troubleshooting access issues, and verifying successful access for authorized users in an Active Directory environment.

Environment

  • Windows Server 2022
  • Active Directory
  • Security Groups
  • SMB Shares
  • NTFS Permissions

Video 1 - Creating and Sharing the HR Folder

Created the HR folder and configured SMB sharing so that network users could access the resource.

Video 2 - Configuring NTFS Permissions

Configured file system permissions to ensure that only authorized users would have access to the folder.

Video 3 - Investigating Security Group Membership

Investigated group membership and verified that the affected user had not been added to the correct security group.

Video 4 - Access Denied Verification

Verified that unauthorized users were prevented from accessing the HR share, demonstrating the effectiveness of the configured permissions.

Video 5 - Successful Access Verification

Confirmed that authorized users could successfully access the share and create files within the folder.

Root Cause

The issue was caused by incorrect security group membership. Because the user was not a member of the appropriate Active Directory group, access to the HR share was denied.

Solution

Verified group membership and corrected the permissions configuration, ensuring that authorized users had the appropriate access while unauthorized users remained restricted.
